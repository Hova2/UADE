select
Obtiene filas de la base de datos y permite realizar la selección de una o varias filas o columnas de una o varias tablas. 

SELECT lista de columnas
[ INTO tabla destino ] 
FROM orígen de datos 
[ WHERE condición ] 
[ GROUP BY agrupamiento ] 
[ HAVING condición del agrupamiento ] 
[ ORDER BY ordenamiento [ ASC | DESC ] ]
SELECT Especifica las columnas que va a devolver la consulta.
INTO Crea una nueva tabla e inserta en ella las filas resultantes de la consulta.
FROM Especifica las tablas o vistas de las que se van a obtener filas.
WHERE Especifica una condición de búsqueda para restringir las filas que se van a devolver.
GROUP BY Especifica los grupos en los que se van colocar las filas de salida.
HAVING Especifica una condición de búsqueda de un grupo.
ORDER BY Especifica el orden del conjunto de resultados.

select (cláusula)
Especifica una lista de columnas que va a devolver la consulta. Pueden estar definidas por nombres de columnas de tablas o vistas especificadas en la cláusula from, o representar expresiones, constantes, variables, funciones; o cualquier combinación de éstas. 

SELECT [ ALL | DISTINCT ] [ TOP n [PERCENT] ] 
    {    * 
        | { tabla | vista | alias }.* 
        |     { columna | expresión  } 
            [ [ AS ] alias_columna ] 
        | alias_columna = expresión 
    }    [ ,...n ]
ALL Especifica que pueden aparecer filas duplicadas en el conjunto de resultados. ALL es el valor predeterminado.
DISTINCT Especifica que sólo pueden aparecer filas exclusivas en el conjunto de resultados. Los valores NULL se consideran iguales a efectos de la palabra clave DISTINCT.
TOP n Especifica que sólo se van a utilizar las primeras n filas del conjunto de resultados de la consulta; n es un valor entero.
PERCENT Si se especifica PERCENT, sólo se va a utilizar el primer n por ciento de filas del conjunto de resultados. Cuando se especifica con PERCENT, n debe ser un entero entre 0 y 100.
tabla | vista |alias Nombre de la tabla o vista de donde se obtienen los datos.
columna Es el nombre de una columna que se va a devolver.Si existen nombre duplicados en distintos orígenes, la referencia al nombre de la columna debe estar calificado.
expresión Es un nombre de columna, constante, función, o una combinación de nombres de columnas, constantes y funciones conectados mediante operadores, o una subconsulta.
AS alias_columna Es un nombre alternativo para reemplazar el nombre de la columna en el conjunto de resultados de la consulta.
Ejemplos: 

-- proyecta la totalidad de la tabla authors.
select * from authors
GO

-- proyecta todas las filas de las columnas city, state.
select all city, state from authors
GO

-- excluye las filas duplicadas de la proyección.
select distinct city, state from authors
GO

-- agrega una expresión y define un encabezado.
select title, price, royalty, price*royalty/100 AS derechos from titles
GO

-- idem anterior.
select title, price, royalty, derechos=price*royalty/100  from titles
GO

-- proyecta las 3 primeras filas de la tabla.
select top 3 title, price, royalty from titles
GO

from
Especifica las tablas, vistas, tablas derivadas y tablas combinadas que se utilizan en las instrucciones DELETE, SELECT y UPDATE.

FROM { tabla [ [ AS ] alias ] ] | vista [ [ AS ] alias ] ] } [, n...]
tabla Es el nombre de una tabla que se va a utilizar en la instrucción.
Si la tabla o la vista existe en otra base de datos del mismo equipo que ejecuta Microsoft® SQL Server™, utilice un nombre totalmente calificado en la forma database.owner.object_name. Si la tabla o la vista se encuentra fuera del servidor local en un servidor vinculado, utilice un nombre de cuatro partes en el formato linked_server.catalog.schema.object.
vista Es el nombre de una vista. Una vista es una "tabla virtual", normalmente creada como un subconjunto de las columnas de una o más tablas.
alias Es un alias para la tabla o vista que se utiliza por comodidad o para distinguirlas en una autocombinación o en una subconsulta. El alias suele ser un nombre de la tabla abreviado, que se utiliza para hacer referencia a columnas específicas de las tablas en una combinación. Si existe el mismo nombre de columna en más de una tabla de la combinación, SQL Server requiere que el nombre de la columna esté calificado por el nombre o el alias de la tabla.
(No se puede utilizar el nombre de la tabla si se ha definido un alias).
Ejemplos de utilización del nombre calificado con y sin definición de un alias. 

-- sin alias
select publishers.pub_id, pub_name, title 
from publishers, titles
GO
-- con alias
select p.pub_id, pub_name, title 
from publishers p, titles
GO

where
Especifica una condición de búsqueda para restringir las filas que se van a devolver. 

WHERE { [ NOT ] < expresión_booleana > 
                [ { AND | OR } ] 
                [ NOT ] < expresión_booleana > [ ... ] }
where Limita las filas devueltas en el conjunto de resultados mediante el uso de predicados. No hay límite en el número de predicados que se pueden incluir en una condición de búsqueda.
expresión_lógica Cualquier combinación entre variables, constantes, funciones y operadores de comparación o lógicos que devuelvan un valor booleano TRUE , FALSE o UNKNOWN.
AND | OR Operadores lógicos que combinan expresiones booleanas. Cuando en una expresión se utiliza más de un operador lógico, primero se evalúan los operadores AND y luego los operadores OR. Se puede cambiar el orden de la evaluación si se utilizan paréntesis.
NOT Niega un predicado. NOT invierte el valor de cualquier expresión booleana.
Detalle de las editoriales y los libros publicados por éstas. 

select p.*, t.*
from publishers p, titles t
where p.pub_id = t.pub.id
GO
Nombre de los autores y libros de los mismos sobre negocios o psicología. 

select au_lname, au_fname, title, type
from authors a, titles t, titleauthor ta
where a.au_id = ta.au_id
    and t.title_id = ta.title_id
    and (type = 'business' or type = 'psychology')
GO

group by
Especifica los criterios de agrupamiento para obtener valores totales a cada grupo obtenido.

[ GROUP BY { columna1 [ ,...columnaN] }

Los valores NULL se tratan como los valores más bajos posibles.
columna Es el criterio por el cuál se agrupa, generando un nuevo grupo por cada valor distinto. Si se especifica mas de una columna, el segundo criterio determina un nuevo grupo si el anterior es el mismo. Los valores de totales se obtienen mediante las funciones count(), max(), min(), sum() o avg().

order by
Especifica el orden del conjunto de resultados.

[ ORDER BY { expresión [ ASC | DESC ] }  [ ,...n] ]
expresión Especifica la columna según la que se ordenará. Se puede especificar una columna de orden como un nombre o alias de columna (que puede estar calificado con el nombre de una tabla o vista), una expresión o un entero no negativo que representa la posición del nombre, alias o expresión en la lista de selección.
Se puede especificar varias columnas de orden. La secuencia de columnas de orden en la cláusula ORDER BY define la estructura del conjunto ordenado de resultados.
ASC Indica que los valores de la columna especificada se deben ordenar de manera ascendente, desde el valor más bajo al más alto.
DESC Indica que los valores de la columna especificada se deben ordenar de manera descendente, desde el valor más alto al valor más bajo.
Los valores NULL se tratan como los valores más bajos posibles.
Nombre de los autores y libros de los mismos ordenado por título y autor. 

select au_lname, au_fname, title
from authors a, titles t, titleauthor ta
where a.au_id = ta.au_id and t.title_id = ta.title_id
order by title desc, 2, 3
GO
