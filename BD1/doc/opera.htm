Operadores de conjunto

Son operadores lógicos que comprueban la veracidad de alguna condición. Devuelven el tipo de datos Boolean con valor TRUE o FALSE.

Operador

Significado

ALL

TRUE si el conjunto completo de comparaciones es TRUE.

ANY

TRUE si cualquier miembro del conjunto de comparaciones es TRUE.

EXISTS

TRUE si una subconsulta contiene a cualquiera de las filas.

IN

TRUE si el operando es igual a uno de la lista de expresiones.

ALL

Compara un valor escalar con un conjunto de valores de una sola columna.

Sintaxis

expresión escalar { = | <> | != | > | >= | !> | < | <= | !< } ALL ( subquery )

Argumentos

expresión escalar

Es cualquier expresión válida de Microsoft® SQL Server™.

{ = | <> | != | > | >= | !> | < | <= | !< }

Es un operador de comparación.

subquery

Es una subconsulta que devuelve un conjunto de resultados de una columna. El tipo de datos de la columna devuelta debe ser el mismo tipo de datos que el de la expresión escalar.

Tipos devueltos

Boolean

Valor del resultado

Devuelve TRUE cuando la comparación especificada es TRUE para todas las parejas (expresión escalar, x) donde x es un valor del conjunto de columna única; de lo contrario, devuelve FALSE.

ANY

Compara un valor escalar con un conjunto de valores de una sola columna.

Sintaxis

expresión escalar { = | < > | ! = | > | > = | ! > | < | < = | ! < } { ANY } ( subquery )

Argumentos

expresión escalar

Es cualquier expresión válida de Microsoft® SQL Server™.

{ = | <> | != | > | >= | !> | < | <= | !< }

Es cualquier operador de comparación válido.

ANY

Especifica que debe hacerse una comparación.

subquery

Es una subconsulta cuyo conjunto de resultados tiene una columna. El tipo de datos de la columna devuelta debe coincidir con el de la expresión escalar..

Tipos de resultado

Boolean

Valor del resultado

ANY devuelve TRUE cuando la comparación especificada es TRUE para una pareja (expresión escalar, x) donde x es un valor del conjunto de resultados de una sola columna. De lo contrario, devuelve FALSE.

EXISTS

Especifica una subconsulta para probar la existencia de filas.

Sintaxis

EXISTS subquery

Argumentos

subquery

Es una instrucción SELECT restringida (la cláusula COMPUTE y la palabra clave INTO no están permitidas.

Tipos de resultado

Boolean

Valores de resultado

Devuelve TRUE si alguna subconsulta contiene alguna fila.

IN

Determina si un valor dado coincide con algún valor de una subconsulta o lista.

Sintaxis

expresión [ NOT ] IN
    (
        subquery
        | expresión [ ,...n ]
    )

Argumentos

expresión

Es cualquier expresión válida de Microsoft® SQL Server™.

subquery

Es una subconsulta cuyo conjunto de resultados tiene una columna. Esta columna debe tener el mismo tipo de datos que la expresión

expresión [,...n]

Se trata de una lista de expresiones en la que se buscará una coincidencia. Todas las expresiones deben ser del mismo tipo que la expresión.

Tipos de resultado

Boolean

Valor del resultado

Si el valor de la expresión es igual a cualquier valor devuelto por subquery o es igual a cualquier expresión de la lista separada por comas, el valor del resultado es TRUE. En caso contrario, el valor del resultado es FALSE.

Con NOT IN se niega el valor devuelto.
