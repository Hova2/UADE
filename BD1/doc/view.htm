alter view
ALTER VIEW nombre [ ( columna [ ,...n ] ) ]
AS 
sentencia_select
[ WITH CHECK OPTION ]
nombre Es la vista que va a cambiar.
No afecta a los procedimientos almacenados ni a los desencadenadores dependientes, ni a los permisos establecidos.
columna Es el nombre de una o más columnas, separadas por comas, que van a formar parte de la vista .
sentencia La instrucción select que defina la vista.
with check option Impone que todas las instrucciones de modificación de datos ejecutadas contra la vista se adhieran a los criterios establecidos en la sentencia select que define la vista.
Modifica una vista creada anteriormente. 
USE pubs
ALTER VIEW libros_por_autor ([titulo del libro], [apellido del autor],
[nombre del autor]
AS
SELECT title, au_lname, au_fname
FROM ( titles t INNER JOIN titleauthor ta ON t.title_id = ta.title_id )
	INNER JOIN authors a ON a.au_id = a.au_id
GO

create view
CREATE VIEW nombre [ ( columna [ ,...n ] ) ]

AS 
sentencia_select
[ WITH CHECK OPTION ]
nombre Es el nombre de la vista. Los nombres de las vistas deben cumplir las reglas de los identificadores. Especificar el propietario de la vista es opcional.
Para crear una vista, el usuario debe tener permisos CREATE VIEW y permisos SELECT en las tablas, vistas y funciones de valores de tabla a las que se hace referencia en la vista, y permisos EXECUTE en las funciones de valores escalares a las que se llama en la vista.
columna Es el nombre que se va a utilizar para una columna en una vista. Sólo es necesario asignar un nombre a una columna en CREATE VIEW cuando una columna proviene de una expresión aritmética, una función o una constante; cuando dos o más columnas puedan tener el mismo nombre (normalmente, debido a una combinación); o cuando una columna de una vista recibe un nombre distinto al de la columna de la que proviene. Los nombres de columna se pueden asignar también en la instrucción SELECT.
Si no se especifica el nombre de lacolumna, éstas adquieren los mismos nombres que las columnas de la instrucción SELECT.
sentencia select Es la instrucción SELECT que define la vista. Puede utilizar más de una tabla y otras vistas. Para seleccionar los objetos a los que se hace referencia en la cláusula SELECT de una vista creada, es necesario tener los permisos adecuados.
with check option Exige que todas las instrucciones de modificación de datos ejecutadas contra la vista se adhieran a los criterios establecidos en select_statement. Cuando una fila se modifica mediante una vista, WITH CHECK OPTION garantiza que los datos permanecerán visibles en toda la vista después de confirmar la modificación.
Crea una vista utilizando un bloque case para proyectar los datos. 
CREATE VIEW vw_elementos
AS
SELECT elemento, 
    color = case codcolor when 'v' then 'verde'
                                    when 'r' then 'rojo'
                                    else 'otro' end					 	
FROM elementos
GO
Crea una vista definiendo los nombres de las columnas en la cabecera del create. 
USE pubs
CREATE VIEW lista_de_precios ([titulo del libro], tipo, [precio de lista])
AS
    SELECT title, type, price FROM titles
GO
Crea una vista reuniendo tres tablas mediante la cláusula JOIN. 
USE pubs
CREATE VIEW lbros_por_autor
AS
SELECT libro=title, apellido=au_lname, nombre=au_fname
FROM ( titles t INNER JOIN titleauthor ta ON t.title_id = ta.title_id )
	INNER JOIN authors a ON a.au_id = a.au_id
GO
