insert into values
Agrega una fila a una tabla o a una vista. 

INSERT [ INTO ] { nombre_tabla | nombre_vista [ ( nombre_columna, ...n ) ] 
    VALUES ( { expresion | null } [ , ...n ] ) }
INTO Palabra opcional a continuación del INSERT.
nombre_tabla Es el nombre de la tabla que va a recibir los datos.
nombre_vista Se trata del nombre de una vista. La vista a la que hace referancia debe ser actualizable.
nombre_columna Se trata de una lista de una o más columnas en las que se insertarán los datos; debe estar entre paréntesis y delimitada con comas.
Si una columna no está en la lista, se debe proporcionar un valor basado en la definición de la columna; en caso contrario, no se puede cargar la fila.
SQL Server proporciona automáticamente un valor para la columna si ésta:
  • Tiene una propiedad IDENTITY. Se utiliza el siguiente valor de identidad incremental.
  • Tiene un valor predeterminado. Se utiliza el valor predeterminado de la columna.
  • Acepta valores NULL. Se utiliza un valor NULL.
VALUES Presenta la lista de valores de datos que se van a insertar.
  • Debe haber un valor de datos para cada columna definida en la lista de columnas (si se especificó) o en la tabla.
  • La lista de valores debe estar entre paréntesis.
  • Debe respetar el orden definido en la lista de columnas, sino el de la tabla.
expresion Es una constante, variable o expresión que no contenga una instrucción SELECT o EXECUTE.
null Unicamente si la columna admite valores nulos.
Ejemplo que agrega un fila compuesta por una columna varchar(n) y una columna char(n), a la tabla Proveedores. 

INSERT INTO Proveedores(nombre, cuit)
VALUES('PROIND SRL', '20-25890012-1')
GO
Ejemplo que agrega un fila compuesta por un char(n), un smallint y un smalldatetime, a la tabla Compras. 

INSERT INTO Compras(cuit, nro_compra, fecha)
VALUES('20-25890012-1', 1405, '20030624')
GO

insert into select
Agrega una o mas fila(s) a una tabla o a una vista. 

INSERT [ INTO ] { destino [ ( nombre_columna, ...n ) ] 
    SELECT { lista de expresiones | * } FROM origen  
                    [ WHERE ...] [ GROUP BY ...] [ HAVING ...]
destino Nombre de la tabla o vista donde se insertarán los datos.
nombre_columna Ver detalle de la instrucción INSERT INTO.
lista de expresiones Presenta la lista de valores de datos que se van a insertar obtenidos de una proyección de valores mediante la instrucción SELECT.
  • Debe haber un valor de datos para cada columna definida en la lista de columnas (si se especificó) o en la tabla destino.
  • Debe respetar el orden definido en la lista de columnas, sino el de la tabla destino.
origen Nombre de la tabla o vista de donde se obtienen los datos.
Este ejemplo agrega a la Tabla Compras_Año_2002 las filas proyectadas de la tabla Compras correspondientes al año 2002. 

INSERT INTO Compras_Año_2002(cuit, nro_compra, fecha)
SELECT cuit, nro_compra, fecha FROM compras WHERE year(fecha) = 2002;
GO

delete
Quita filas de una tabla. 

DELETE 
    [ FROM ] tabla_destino
        [ FROM { < tabla_origen > } [ ,...n ] ] 
    [ WHERE < condición > ]
tabla_destino Es la tabla en donde se quitarán las filas.
tabla_origen La cláusula opcional FROM tabla_origen especifica una combinación, en lugar de una subconsulta en la cláusula WHERE para identificar las filas que se van a quitar.
WHERE Especifica las condiciones utilizadas para limitar el número de filas que se van a eliminar. Si no se proporciona una cláusula WHERE, DELETE quita todas las filas de la tabla.
Elimina todas las filas de la tabla. 

   DELETE  titleauthor
Elimina las filas de la tabla que cumplan con la condición. 

   DELETE  titleauthor WHERE id_autor = 200

update
Cambia datos de una tabla. 

UPDATE { tabla_destino }
    SET 
        { columna = { expresión | DEFAULT | NULL } } [ ,...n ] 
    [ FROM { < tabla_origen > } [ ,...n ] ] 
    [ WHERE < condición > ]
tabla_destino Es la tabla en donde se van a actualizar las filas.
SET Especifica la lista de nombres de columnas o variables que se van a actualizar.
columna Es una columna que contiene los datos que se van a cambiar. Se debe encontrar en la tabla o en la vista especificadas en la cláusula UPDATE.
expresión Es una variable, un valor literal, una expresión o una instrucción subSELECT entre paréntesis que devuelve un solo valor. El valor devuelto sustituye al valor existente en la columna o @variable.
default Especifica que el valor predeterminado definido para la columna debe sustituir al valor existente en la columna. También se puede utilizar para cambiar la columna a NULL si no tiene valor predeterminado y en su definición se especifica que puede aceptar valores nulos.
tabla_origen La cláusula opcional FROM tabla_origen especifica una combinación, en lugar de una subconsulta en la cláusula WHERE para identificar las filas que se van a cambiar.
WHERE Especifica las condiciones utilizadas para limitar el número de filas que se van a modificar. Si no se proporciona una cláusula WHERE, UPDATE cambia las columnas especificadas de todas las filas de la tabla.
Modifica la columna fecha de todas las filas de la tabla. 

   UPDATE Compras
      SET fecha = getDate()
Modifica la columna fecha de las filas de la tabla que cumplan con la condición. 

   UPDATE Compras
      SET fecha = getDate()
      WHERE nro_compra = 1056
